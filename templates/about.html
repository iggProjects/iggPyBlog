My experience in the course

First of all, I must emphasize that the facility where we received the course was up to the challenge. On the other hand, the professor is a young person, who enjoys teaching, and who is concerned about keeping up with the news related to Python and the emerging programming web environments.

The program covered many topics, which does not make it easy to understand some core elements. For example, spending only three days on object-oriented programming only allows you to see a small part of that world.

It is not easy for a professor to teach a course where there are different levels of programming and systems knowledge, and most likely, different expectations about the program.

In my particular case, I think I was able to meet the expectations that I set out to develop in the course. In addition, I had the support of the people at pythonanywhere, who quickly answered my questions about running code in the background.

In short, I was satisfied with the relationship between my expectations and the product I developed.

My Bitakora

Three years ago I worked on a exercise using python. I did it with a Venezuelan friend. We set out to make the game of life using python. In order not to forget the path I have taken, I enrolled in the introductory course to Python offered by Lanbide through the Nazareth Academy.

I had three ideas before the course, to choose from. The first was to make a database client using Python. The second, to make an app to evaluate the financial performance of leading companies. The third, to make a web page that, on the one hand, would be a kind of course log, in which some of the exercises carried out are presented, while on the other hand, it would develop a kind of console client to execute python programs, including an extra, made up of the Game of Life, and three exercises using Python's multithreading option.

The truth is that at first the challenge seemed demanding to me a lot, since having very little knowledge of Python, I would have to resort to the experience acquired in the course given by Alberto Mozo in Aulafor, where I developed a client in PHP to manage mysql-mariadb, a very simple version of phpmyadmin.

To keep a daily record of what was discussed in the course, I used Google Drive to record each day's notes and my contribution to the topic.

'Bitakora.doc' also includes links to resources that I found interesting to delve into the points treated in class.

For reasons of time, the design of the web page emulates a blog, in which the content of the posts is inside several list of dictionaries related to the articles, excercises, LifeGame and Enigma tabs (article_data.py, excercise_data.py, EnigmaGame_data.py and LifeGame_data.py). Additionally, there is a tab for the sqlite based example.

Some fields of the dictionary are 'url' addresses, which will be called in the internal pages to which the navigation leads.

Another element of the design was to include three buttons to execute the python code related to the requested exercise and a button to access the code. The first option is to download a zip file containing the python code and a '.bat' file to run it from the folder where they are downloaded. The second option simulates a web console, in which you will see the execution of the script. And the third option is to run the script in the VSC console itself, used to activate the flask_app.py application.

To simulate the 'web console', I first run the selected python code in the background, and then, each line of the result is interpreted, so that it can be drawn on the web page via javascript.

Finally, I must comment that in order to document the obstacles that were being presented and the tasks to be done, two files are included in Github, 'z-bitakora.txt' and 'z-ToDo-date.txt'. They present the topics and links consulted for the search for solutions, and the tasks to be completed by a certain date.

Five important things remain pending: 1.- in the 'web console', simulate the python 'input command', in order to be able to run codes that ask the user for data before executing the operations; 2.- Create the function to write the warnings and/or errors in log files, in order to prevent them from appearing on the user's screen, except in the case of 'fatal errors', in which a friendly message is placed for the user explaining the situation and what to do in those cases; 3.- Use of "progressive bar" when click 'HTML button' to execute the py code in web page; 4.- Improve the css style to make the page really responsive; 5.- incorporate multi-language option.

Suggestions

This course has a very extensive program, and given its duration (100 hours), it implies that it is not possible to delve into the points that are being touched on. It would be interesting to divide it into three courses: 1.- algorithms in Python; 2.- database management and web development in Python; 3.- data analysis and automation tasks.

The three courses would be introductory, since on the one hand, Python is an open source language that is growing a lot on the market, and on the other, the topics themselves are very extensive.

As for the quizzes, I suggest that instead of doing just one at the end, you do a quiz every time a topic in the program is finished.

Both Nazareth Academy and Professor Kulhan would have an excellent opportunity to support Lanbide in bringing young people and professionals closer to Big Data, Machine Learning and Automation Tasks using Python.
